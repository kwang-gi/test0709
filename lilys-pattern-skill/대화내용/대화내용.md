# LilysAI 패턴 조사 및 스킬 제작 - 대화 요약

## 1. 조사 배경
LilysAI(릴리스AI)의 유튜브 영상 요약 페이지를 대상으로, "요약"과 "확장" 탭에서 생성되는 콘텐츠가 원본 스크립트(자막) 내용만을 근거로 만들어지는지, 아니면 릴리가 별도로 웹 검색을 수행해 외부 정보를 추가하는지를 확인하는 것이 목적이었다.

## 2. 확인된 핵심 패턴

### 2-1. 요약(4종) 탭
- 짧게 / 기본 / 길게 / 쉽게 4가지 옵션이 있으며, 전부 원본 스크립트 내용을 압축·재구성한 것으로 확인됨. 각 문장에는 스크립트의 타임스탬프와 연결된 인용번호(예: [24], [33])가 달려 있고, 클릭하면 좌측 스크립트 패널의 해당 구간이 하이라이트됨.
- 네 가지 길이/난이도 옵션 모두 별도의 외부 검색 없이 스크립트 내부 정보만으로 생성되는 것으로 판단됨.

### 2-2. 확장(6~8종) 탭
- 인포그래픽, 액션아이템 도출, 플래시카드, 퀴즈 등 대부분은 스크립트 기반으로 생성됨.
- 다만 일부 항목(배경 지식 설명, 반대 의견/관점 제공형 리포트 등)은 원문에 없는 개념 설명이 필요할 때 별도 웹 검색을 수행하는 것으로 확인됨.
- "확장 리포트 추가" 모달에서 템플릿 카테고리를 전수 확인했고, "나만의 템플릿 만들기" 기능은 이름과 프롬프트 두 개의 입력 필드만 제공하는 단순한 커스텀 템플릿 생성 도구임을 확인함.

### 2-3. 채팅(LILY) 탭 검색 패턴
- 채팅창 전체를 처음부터 끝까지 스크롤하며 모든 질문/답변을 순서대로 전수 조사함.
- 스크립트 안에 있는 내용에 대한 질문에는 검색을 하지 않고 스크립트 내용만으로 답변함.
- 스크립트 범위를 벗어난 외부 개념이나 배경지식을 묻는 질문에서만 웹 검색이 1회 발동되는 패턴을 확인함(예: "🌐 'Hermes Agent Minist...' 검색 완료 · 6개 자료" 형태의 검색 마커가 표시됨).
- 답변 말미에 항상 후속 질문 3가지를 자동으로 추천하는 패턴이 반복적으로 나타남.

### 2-4. 애니메이션(컴퓨터 제어) 관련
- 확장 탭의 "애니메이션" 옵션이 실제로 컴퓨터를 직접 제어하는 기능은 아니며, 별도의 시각화/애니메이션 콘텐츠를 생성하는 확장 리포트 유형 중 하나임을 확인함.

### 2-5. /learn 명령어
- 사용자가 제공한 강의 스크립트 내 /learn 명령어 설명을 바탕으로, 이를 Claude(에이전트) 환경에서 어떻게 적용할 수 있는지 논의함.

## 3. 결론 및 목표
위 패턴을 재현 가능한 규칙으로 정리하여, LilysAI 사이트에 직접 방문하지 않고도 스크립트만 주어지면 동일한 형태(요약 4종 + 확장 6~8종 + 커스텀 템플릿 + 후속 질문 3가지 규칙)로 정리해주는 스킬 패키지를 만들기로 함.

## 4. 최종 실행 지시 사항 (사용자 원문 요약)
- 커스텀 옵션("나만의 템플릿 만들기")도 스킬에 포함할 것.
- 중간 그림(인포그래픽 등)은 "코덱스 이미지 2.0" 스킬을 호출해 생성하되, 정확한 호출 방식은 사용자가 나중에 직접 수정할 것이므로 placeholder 형태로만 넣어둘 것.
- GitHub 저장소(https://github.com/kwang-gi/test0709)에 폴더 1개(lilys-pattern-skill)만 만들고 그 안에 모든 것을 넣을 것.
- 설명 문서는 HTML로 작성하고, 왼쪽 메뉴 + 카테고리별 정리 + 흰색 배경으로 구성할 것 (index.html).
- 대화 내용은 별도 폴더(대화내용/)를 만들고 그 안에 대화내용.md 파일 하나로 전부 저장할 것.
- 스킬 자체는 별도 폴더(skill/)를 만들어 그 안에 SKILL.md, templates/, run.py 등을 넣을 것.

## 5. 생성된 최종 폴더 구조
```
lilys-pattern-skill/
├── index.html                          (설명 문서: 왼쪽 메뉴 + 카테고리별 정리 + 흰 배경)
├── 대화내용/
│   └── 대화내용.md                      (본 문서)
└── skill/
    ├── SKILL.md
    ├── run.py                           (Codex 이미지 2.0 호출 placeholder 포함)
    └── templates/
        ├── summary_short.md
        ├── summary_basic.md
        ├── summary_long.md
        ├── summary_easy.md
        ├── ext_action_items.md
        ├── ext_infographic.md
        ├── ext_flashcard.md
        ├── ext_quiz.md
        ├── ext_background.md            (웹검색 기반)
        ├── ext_counterview.md           (웹검색 기반)
        ├── ext_custom.md                (커스텀 템플릿)
        └── followup_questions.md        (후속 질문 3가지 규칙)
```

## 6. 참고 사항
- Codex 이미지 2.0 스킬 호출부는 run.py 내에 `invoke_skill("codex-image-2.0", ...)` 형태의 일반 placeholder로 작성되어 있으며, 실제 PC에 설치된 스킬 이름/명령으로 사용자가 직접 교체해야 함.
- 본 문서는 조사 및 작업 과정을 기록한 대화 요약본이며, 원본 대화 전체의 세부 스크린샷·검증 과정은 생략하고 핵심 결론 위주로 정리함.
