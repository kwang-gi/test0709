LilysAI 패턴 재현 스킬

유튜브 스크립트 하나만 주면 릴리처럼 요약 4종 + 확장 리포트 + 추천질문 3개를 자동 생성하는 스킬입니다. 릴리에 접속하지 않고 Claude / Claude Code에서 동일한 정리물을 만듭니다.

1. 이 프로젝트가 뭔가요

LilysAI(릴리스AI)가 유튜브 영상을 요약·확장하는 방식을 관찰·역설계하여, 같은 결과를 로컬에서 재현할 수 있게 만든 스킬 패키지입니다. 핵심 발견은 릴리의 모든 요약/확장이 내부적으로 [스크립트 전문] + [고정 프롬프트] -> LLM -> 포맷된 결과물 구조라는 점입니다. ("나만의 템플릿 만들기"가 이름 + 프롬프트 딱 두 입력만 받는 것으로 확인됨.)

2. 폴더 구조

lilys-pattern-skill/
|- index.html              (이 설명 문서)
|- 대화내용/
|   |- 대화내용.md          (지금까지의 전체 대화 기록)
|- skill/
    |- SKILL.md            (스킬 정의/진입점)
    |- templates/          (요약/확장/추천질문 프롬프트들)
    |- run.py              (실행 스크립트, 프레임 추출 옵션 포함)

3. 요약 4종 규칙

짧게 · 기본 · 길게 · 쉽게 - 이 넷은 정보 출처가 아니라 압축률과 말투만 다릅니다. 넷 다 스크립트 안에서만 생성되며, 모든 문장에 스크립트 타임스탬프 인용 [숫자]가 붙습니다.

옵션특징
짧게핵심 불릿 몇 개로 최대 압축
기본표준 요약 (소제목 + 불릿)
길게소제목 + 세부 항목(a,b,c)까지 촘촘하게
쉽게반말·비유체로 쉽게 풀어씀

4. 확장 리포트 규칙

확장은 카테고리별 고정 프롬프트 템플릿입니다.

  • 더 깊이 이해하기 관련 배경지식, 반대 시각 조사, 액션아이템 도출, 댓글분석
  • 시각자료 애니메이션, 인포그래픽
  • 이해 점검 플래시카드, 퀴즈, 팟캐스트
  • 문서형식 교재, 보고서, 회의록, 사용자 인터뷰, 문헌검토, 논문 리뷰 등

5. 커스텀 옵션

릴리의 "나만의 템플릿 만들기"와 동일하게, 이름 + 프롬프트만 정의하면 새 확장을 추가할 수 있습니다. templates/ext_custom.md를 복제해서 프롬프트만 바꾸면 됩니다.

6. 추천질문 3개 규칙

매 답변 뒤에 붙는 추천질문 3개는 직전 답변의 하위 주제를, 사용자 1인칭 반말("~알려줘", "~보여줘")로 생성합니다. templates/followup_questions.md 참고.

7. 검색이 걸리는 경우

스크립트 안에서 답할 수 있으면 검색하지 않습니다. 스크립트에 없는 새 외부 개념/고유명사가 처음 등장할 때만 웹 검색을 1회 수행하고, 이후 후속 질문은 확보한 자료+스크립트로 답합니다. "관련 배경지식", "반대 시각 조사" 템플릿에만 웹검색 도구를 연결하세요.

8. 화면 그림 처리

자막에 없고 화면에만 나오는 다이어그램은 텍스트만으론 복원 불가입니다. run.py가 장면 전환 프레임을 추출 후 비전 모델로 설명을 붙입니다. 새 그림 생성은 Codex 이미지 2.0 스킬을 호출하도록 자리를 비워뒀습니다(SKILL.md에서 나중에 직접 수정).

9. 스킬 사용 방법

# Claude Code에서 skill 폴더를 프로젝트에 두고:
"이 유튜브 스크립트를 lilys 패턴으로 정리해줘"

# 또는 스크립트로
python skill/run.py --input script.txt --outdir ./output

10. 파일 목록

  • skill/SKILL.md - 스킬 진입점 / 전체 규칙
  • skill/templates/summary_*.md - 요약 4종
  • skill/templates/ext_*.md - 확장 리포트들 + 커스텀
  • skill/templates/followup_questions.md - 추천질문 3개
  • skill/run.py - 실행 스크립트
  • 대화내용/대화내용.md - 전체 대화 기록
Codex 이미지 2.0 스킬 호출 부분은 일반 형태로 넣어뒀습니다. PC에 있는 실제 스킬 이름으로 나중에 바꿔주세요.
