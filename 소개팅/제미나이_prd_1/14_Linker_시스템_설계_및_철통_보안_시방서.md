# [Linker] 시스템 설계 및 철통 보안 시방서

**작성일:** 2026-04-13
**문서 버전:** v2.0(기술 심화본)

## 1. 문어발 자르기(Failover) 서버 아키텍처
- **대문 서버(Gateway):** 데이터 비저장 방식. 오직 요청 중계만 수행.
- **자폭 프로토콜:** 관리자 패널에서 버튼 클릭 시 즉시 인스턴스 파기 및 스토리지 0 채우기(Zeroing) 실행.
- **자동 우회:** DNS 가중치 조정을 통해 1분 내에 전 세계 예비 서버로 유저 트래픽 자동 전환.

## 2. 유령 데이터베이스(Ghost DB) 보안 로직
- **물리 격리:** 인터넷 망에서 접근 불가한 VPC 전용 서브넷 운영.
- **이중 암호화:** 필드 단위로 암호화 키를 분산 관리하여 DB 관리자조차 내용을 볼 수 없게 처리.
- **HSM 마스터 키:** 암호화의 핵심 키는 하드웨어 보안 장치에 보관되어 물리적 접근 없이는 탈취 불가.

## 3. Tauri 기반 보안 클라이언트 및 P2P 저장
- **Webview 차단:** 일반 브라우저에서 소스 보기가 불가능한 네이티브 바이너리 배포.
- **사생활 보호:** 채팅 데이터는 서버에 남기지 않고 클라이언트 로컬 스토리지에 암호화 저장. 서버 압수수색 시에도 유저 대화 내용은 절대 확인 불가.

## 4. 등기 인증(PAV) 기술 연동
- **주소 API 연동:** 실명 확인 시 주소 데이터를 수집하여 등기 발송 자동화 시스템 구축.
- **보안 코드 검증:** 일회성 보안 코드(OTAC) 발급 및 수령 여부 교차 검증 로직 구현.
