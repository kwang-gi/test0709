# Linker(가칭) 소스 코드 보안 및 관리 가이드(Security Policy)

**작성일:** 2026-04-13
**문서 버전:** 1.1(정산 및 금융 보안 강화)
**문서 목적:** AI 협업 개발 및 자체 개발 시 소스 코드 유출을 방지하고, 핵심 비즈니스 로직 및 자산을 안전하게 보호하기 위한 표준.

## 1. 버전 관리 및 소셜 저장소 보안(Git & Private Repo)
- **Private Repository 필수:** 모든 코드는 비공개로 운영하며 대표님 외 접근을 철저히 통제합니다.
- **2단계 인증(2FA):** 저장소 계정 탈취를 막기 위해 물리적 보안 키나 생체 인증을 병행합니다.

## 2. 민감 정보 분리 관리(.env 시스템)
- **환경 변수 관리:** DB 주소, 결제 API 키 등은 코드와 완전히 분리하여 관리합니다.
- **메모리 주입:** 서버 구동 시 비밀번호는 디스크가 아닌 메모리에만 존재하도록 설정하여 서버가 꺼지면 흔적조차 남지 않게 합니다.

## 3. AI 협업 시 보안 주의사항(AI Coding Safety)
- **익명화 질문:** 로직을 물어볼 때 실제 비즈니스 수치나 키값은 무조건 가상의 데이터로 대체합니다.

## 4. 소스 코드 암호화 및 난독화(Obfuscation)
- **Tauri 바이너리화:** 설치 파일 형태로 배포하여 일반적인 웹 브라우저의 소스 보기를 원천 차단합니다.

## 5. 로컬 작업 환경 보안(VeraCrypt 활용)
- **암호화 가상 드라이브:** 작업 중에는 베라크립트 내에서만 소스를 열고, 종료 시 드라이브를 해제하여 물리적 침입에 대비합니다.

## 6. 데이터베이스(DB) 백업 및 생존 전략
- **자동 스냅샷 & 오프사이트 백업:** 주기적으로 암호화된 백업본을 제3의 안전한 지역으로 자동 전송합니다.

## 7. 정산 및 금융 거래 보안(Financial Transaction Security) - [중요]
정산 과정에서 발생할 수 있는 '보안의 틈'을 메우기 위한 핵심 아키텍처입니다.
- **ID 기반 익명 정산:** 정산 시스템 내에서 유저의 실명이나 계좌번호는 직접 노출되지 않습니다. 정산ID: TX-9901과 같은 식별자만 사용하며, 실제 개인정보와의 연결은 대표님의 마스터 키가 있어야만 복호화되는 격리된 보안 존에서만 이루어집니다.
- **웹훅(Webhook) 서명 검증:** 결제사(Toss 등)로부터 오는 모든 상태 변경 신호는 서버 간의 비밀 서명을 대조하여 0.1%의 조작 가능성도 차단합니다.
- **이중 승인 구조(Dual-Approval):** 고액 정산(예: 100만 원 이상)의 경우, 시스템 자동 정산 외에 운영자(대표님)가 별도의 보안 장치(OTAC 등)를 통해 승인해야만 이체가 실행되도록 물리적 허들을 둡니다.
- **수정 불가능한 감사 로그(Immutable Audit Log):** "누가, 언제, 어떤 정산 데이터를 조회했는가"에 대한 기록은 삭제가 불가능한 전용 로그 서버에 기록됩니다.
